# Correção definitiva do Sitemap XML

Analise toda a implementação responsável pela geração do sitemap (Next.js App Router, Pages Router ou qualquer outro método utilizado) e corrija todos os problemas de compatibilidade com o padrão oficial do protocolo Sitemap XML.

## Problema atual

O Google Search Console informa:

* O sitemap pode ser lido, mas contém erros.
* Erro na análise.
* Linha 130: **EntityRef: expecting ';'**

Isso indica que o XML gerado é inválido.

## Objetivo

Gerar um sitemap 100% válido segundo a especificação XML e aceito pelo Google Search Console, Bing Webmaster Tools e demais mecanismos de busca.

## Corrigir obrigatoriamente

1. Localizar qualquer caractere XML inválido, principalmente:

   * &
   * <
   * >
   * "
   * '

2. Escapar corretamente todos os caracteres especiais utilizando:

* & → &
* < → <
* > → >
* " → "
* ' → '

3. Garantir que nenhuma URL contendo parâmetros de consulta (`?`, `&`, etc.) seja escrita diretamente no XML sem a devida codificação.

4. Validar todos os slugs vindos do banco de dados.

5. Garantir que nenhum título, descrição ou texto seja utilizado dentro das tags `<loc>`.

6. Garantir que apenas URLs válidas sejam inseridas.

7. Remover qualquer URL nula, vazia ou inválida.

8. Garantir que todas as tags XML estejam corretamente abertas e fechadas:

* `<urlset>`
* `<url>`
* `<loc>`
* `<lastmod>`
* `<changefreq>`
* `<priority>`

9. Validar o namespace oficial:

```xml
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
```

10. Garantir que o arquivo inicie exatamente com:

```xml
<?xml version="1.0" encoding="UTF-8"?>
```

11. Garantir codificação UTF-8 sem caracteres invisíveis (BOM).

12. Validar que nenhuma URL seja duplicada.

13. Remover URLs que não devem ser indexadas, como:

* /dashboard/*
* /admin/*
* /login
* /register
* /api/*
* páginas privadas
* páginas temporárias

14. Garantir que apenas páginas públicas apareçam no sitemap.

15. Validar que todos os valores de `<priority>` estejam entre 0.0 e 1.0.

16. Validar que `<changefreq>` utilize apenas valores permitidos:

* always
* hourly
* daily
* weekly
* monthly
* yearly
* never

17. Garantir que `<lastmod>` utilize o padrão ISO-8601:

```
2026-07-18T18:52:15.964Z
```

18. Validar o XML final utilizando um parser XML antes de retorná-lo, assegurando que nenhum erro como **"EntityRef: expecting ';'"** ou qualquer outro erro de análise permaneça.

## Importante

Não aplique correções paliativas.

Localize a verdadeira origem do problema na geração do sitemap e corrija a implementação para que o XML produzido seja completamente válido segundo o padrão oficial do Sitemap Protocol.

Ao finalizar, revise todo o sitemap gerado e confirme que ele passa na validação XML sem qualquer erro e é totalmente compatível com o Google Search Console.
